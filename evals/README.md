# Evals

本目录用于持续评估 `opencode-agent-pack` 的分诊与路由质量，重点关注高风险漏判防护。

## 目录
- `rubric.md`：统一评估标准
- `cases/`：代表性任务样例（可持续新增）

## 如何使用
1. 选取一个 case
2. 让系统执行 triage（必要时执行到 review）
3. 对照 case 的“预期结果”与 `rubric.md` 评分
4. 记录偏差类型：
   - 高风险漏判
   - 低风险过度升级
   - lane 错配
   - tier 错配
   - 升级未触发
5. 形成修正动作并更新版本说明

## 新增 case 规范
每个 case 必须包含：
- 任务描述
- 背景
- 预期 complexity/risk/lane
- 预期 tier 路由
- 判断依据
- 错分风险点
- 人工检查要点（可选但建议）

## 评估优先级
最优先不是总体准确率，而是：
**高风险漏判率尽量低**。
