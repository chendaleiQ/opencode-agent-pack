# Skill: receiving-code-review

## Required Behavior

- understand the feedback before changing code
- do not respond with performative agreement
- if feedback is unclear, ask for clarification or report the ambiguity
- fix valid issues directly and minimally
- keep changes inside the approved task boundary

## Guardrails

- do not expand scope just because review mentions adjacent ideas
- do not claim a fix without re-verification
